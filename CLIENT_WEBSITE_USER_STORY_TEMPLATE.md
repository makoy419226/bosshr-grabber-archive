## 9. User Story Backlog

Write stories from the user’s perspective using this format:

> As a **[type of user]**, I want **[action or capability]** so that **[business
> value or outcome]**.

Priorities use MoSCoW:

- **Must:** Required for launch
- **Should:** Important but launch can proceed without it
- **Could:** Optional enhancement
- **Won’t now:** Explicitly outside the current release

### Definition of done

A feature is complete only when:

- Its acceptance criteria pass.
- Mobile and desktop layouts have been tested.
- Required content and links are approved.
- Validation, error, empty, and loading states work.
- Accessibility and basic SEO checks pass.
- No credentials are stored in the repository.
- The client has reviewed and accepted the feature.
